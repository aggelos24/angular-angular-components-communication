3. URL Parameters

A Query string is the part of URL that contains data. We can save variables we want to share as url parameters, and then any component can parse and read them. In Angular, there are route parameters and query parameters; route parameters are not optional and you must define them in route configurations, in contrast to query parameters. For the above reasons, I chose to use query parameters for this example.

Con: User's private data must not be part of the URL.
