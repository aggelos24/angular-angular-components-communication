2. Web Storage

With Web Storage, we can store variables locally in browser. There are two mechanisms to do it, sessionStorage and localStorage. While in sessionStorage data are stored as long as the tab is open, in localStorage data are stored forever. So I chose sessionStorage to use for this example, because there is no way we will end up with junk in our browser local storage.

Cons: 1) We can only store strings, 2) If the components run at the same time, there is no way for the recipient to be notified when the sender sends the message in order to update itself.
