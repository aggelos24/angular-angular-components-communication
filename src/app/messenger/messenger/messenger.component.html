6. Messenger Service

We create a service which has a BehaviorSubject and inject it into both the sender and the recipients. A BehaviorSubject stores the last emitted value from sender. When a recipient subscribes to BehaviorSubject , it receives its current value and every emitted value after that until it unsubscribes. This is an advantage over a Subject, because if we use a Subject and the sender emits a message before the recipient subscribes, the message will be lost.

Con: If there are multiple senders at the same time, some messages may be lost.
