5. Parent-Child Communication (router-outlet)

router-outlet is a directive we place in our code, so Angular will know where to place the corresponding component based on the current URL.

Moreover, router-outlet emits an activate event every time a component is being instantiated, and a deactivate event when it is being destroyed. These events provide access to child component instance, which means that the parent component can change any public variable and call any public function that belongs to the child.

Con: Components communicate only when child is being instantiated or destroyed.
